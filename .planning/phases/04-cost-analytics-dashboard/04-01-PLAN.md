  <behavior>
    **Part A: /api/v1/summary date filtering**

    - summary() with no params → returns all CostEvents (existing behavior unchanged)
    - summary(?from=2024-01-05T00:00:00Z) → returns only events with timestamp >= from
    - summary(?to=2024-01-03T23:59:59Z) → returns only events with timestamp <= to
    - summary(?from=X&to=Y) → both bounds applied
    - summary(?from=not-a-date) → HTTP 400 `{ error: "Invalid 'from' date" }`
    - summary(?to=not-a-date) → HTTP 400 `{ error: "Invalid 'to' date" }`

    **Part B: /api/v1/cost-over-time**

    - cost-over-time(no params, bucket=day) → returns all events grouped by YYYY-MM-DD, no gap fill (no from/to to define bounds)
    - cost-over-time(?from=2024-01-01&to=2024-01-03&bucket=day) → returns 3 entries: 2024-01-01, 2024-01-02, 2024-01-03 — days with no events get cost: 0
    - cost-over-time(?bucket=week) → groups by ISO week Monday date (YYYY-MM-DD of that Monday)
    - cost-over-time(?bucket=month) → groups by YYYY-MM key (e.g., "2024-01")
    - cost-over-time(?from=bad) → HTTP 400 `{ error: "Invalid 'from' date" }`
    - cost-over-time response shape: `{ data: Array<{ date: string; cost: number }>, bucket: string }`

    **Test cases (write these BEFORE implementation):**
    - summary: no params → eventCount = 0 for empty state (already tested, keep)
    - summary: ?from= filters out earlier events correctly
    - summary: ?to= filters out later events correctly
    - summary: ?from=&to= combined — only in-window events counted
    - summary: invalid ?from= → 400
    - summary: invalid ?to= → 400
    - cost-over-time: empty state → { data: [], bucket: 'day' }
    - cost-over-time: 2 events on same day → one bucket with summed cost
    - cost-over-time: events on days 1 and 3, ?from=day1&to=day3 → 3 entries, day 2 has cost: 0
    - cost-over-time: bucket=week → verify Monday grouping
    - cost-over-time: bucket=month → verify YYYY-MM grouping
    - cost-over-time: invalid ?from= → 400
  </behavior>

  <implementation>
    **RED phase:** Write all failing tests in src/server/__tests__/api.test.ts FIRST.
    Add a new `describe('/api/v1/summary date filtering', ...)` block and a new `describe('/api/v1/cost-over-time', ...)` block. Keep existing summary tests intact. Run `bun test src/server/__tests__/api.test.ts` — all new tests must FAIL.

    **GREEN phase:** Update src/server/routes/api.ts:

    1. Update `/summary` route:
       - Extract `fromStr = c.req.query('from')` and `toStr = c.req.query('to')`
       - Validate each: if provided AND `isNaN(new Date(str).getTime())`, return `c.json({ error: "Invalid 'from' date" }, 400)`
       - Filter: `let costs = state.costs; if (from) costs = costs.filter(e => new Date(e.timestamp) >= from); if (to) costs = costs.filter(e => new Date(e.timestamp) <= to)`
       - Rest of aggregation unchanged

    2. Add `/cost-over-time` route after the summary route:
       - Validate ?from/?to same way as summary — return 400 for invalid
       - Filter costs by date range (same filter logic)
       - Bucket loop: for each event, clone date: `const d = new Date(e.timestamp)`. Compute key:
         - `day`: `d.toISOString().slice(0, 10)` → "YYYY-MM-DD"
         - `week`: compute ISO week Monday → `const dow = d.getDay() || 7; d.setDate(d.getDate() - dow + 1); key = d.toISOString().slice(0, 10)`
         - `month`: `key = \`${d.getFullYear()}-${String(d.getMonth()+1).padStart(2,'0')}\``
       - Build `groups = new Map<string, number>()`, accumulate `groups.set(key, (groups.get(key) ?? 0) + e.costUsd)`
       - Gap fill: if `fromStr && toStr && bucket === 'day'`, iterate cursor from `new Date(fromStr)` to `new Date(toStr)`, push `{ date: cursor.toISOString().slice(0,10), cost: groups.get(key) ?? 0 }`. For week/month without range: sort and return groups directly (gap fill only for day bucket within explicit range — week/month gap fill is not required for Phase 4).
       - Return `c.json({ data: result, bucket })`

    **REFACTOR phase:** Extract date validation into a small helper inside api.ts to avoid duplication:
    ```typescript
    function parseDate(str: string | undefined): Date | null | 'invalid' {
      if (!str) return null;
      const d = new Date(str);
      return isNaN(d.getTime()) ? 'invalid' : d;
    }
    ```

    Run `bun test src/server/__tests__/api.test.ts` — all tests must pass.
