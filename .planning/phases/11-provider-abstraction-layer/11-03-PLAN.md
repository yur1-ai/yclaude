  <action>
    **Step 1: Rewrite `src/server/cli.ts`.**

    The current CLI does: parseAll -> applyPrivacyFilter -> computeCosts -> createApp.
    The new CLI does: loadProviders -> createApp.

    Changes:
    a) **Imports**: Replace `import { applyPrivacyFilter, computeCosts, parseAll } from '../index.js'` with `import { loadProviders } from '../providers/registry.js'`. Import types: `import type { ProviderInfo } from '../providers/types.js'`. Keep: `createApp` from `'./server.js'`, `Commander`, `open`, `createRequire`.

    b) **Add --exclude flag** to Commander:
    ```typescript
    .option('--exclude <providers>', 'exclude providers (comma-separated)')
    ```
    Parse it: `const exclude = opts.exclude ? opts.exclude.split(',').map((s: string) => s.trim()) : [];`

    c) **Replace the data pipeline** section. Old:
    ```typescript
    const events = await parseAll(showMessages ? { ...parseOpts, preserveContent: true } : parseOpts);
    const filtered = applyPrivacyFilter(events);
    const costs = computeCosts(filtered);
    ```
    New:
    ```typescript
    const { events, providers } = await loadProviders({
      ...(opts.dir !== undefined ? { dir: opts.dir } : {}),
      preserveContent: showMessages,
      ...(opts.debug ? { debug: true } : {}),
      ...(exclude.length > 0 ? { exclude } : {}),
    });
    ```
    Note: Use conditional spread for `exactOptionalPropertyTypes` safety.

    d) **Replace createApp call**. Old:
    ```typescript
    const app = createApp(
      showMessages
        ? { events: filtered, costs, rawEvents: events, showMessages }
        : { events: filtered, costs, showMessages: false },
    );
    ```
    New:
    ```typescript
    const app = createApp({
      events,
      providers,
      ...(showMessages ? { showMessages } : {}),
    });
    ```

    e) **Add startup banner** with provider status. Per CONTEXT.md: "Compact banner with status icon per provider (checkmark/cross/warning)". Add AFTER the `serve()` callback, before the URL/open messages:

    ```typescript
    serve({ fetch: app.fetch, port, hostname: '127.0.0.1' }, () => {
      console.log(`yclaude v${version}`);
      console.log('');
      // Provider status banner
      for (const p of providers) {
        const icon = p.status === 'loaded'
          ? '\x1b[32m✓\x1b[0m'       // green checkmark
          : p.status === 'not-found'
            ? '\x1b[90m-\x1b[0m'     // gray dash
            : '\x1b[33m!\x1b[0m';    // yellow warning
        const detail = p.status === 'loaded'
          ? `${p.eventCount} events`
          : p.status === 'error'
            ? (p.error ?? 'unknown error')
            : 'not installed';
        console.log(`  ${icon} ${p.name}: ${detail}`);
      }
      console.log('');
      console.log(`Local: \x1b[1m${url}\x1b[0m`);
      // ... rest of open/no-open logic unchanged
    });
    ```

    f) **Debug timing** (--debug shows per-provider detection and load timing per CONTEXT.md). This is already handled inside the registry if debug mode is enabled. The CLI just passes `debug: true` in LoadOptions.

    - Add test for `--exclude` flag parsing: verify it splits comma-separated values correctly
    - Add test for startup banner: mock loadProviders to return ProviderInfo[] with different statuses, verify console output includes status icons

    If the existing tests are minimal (just testing Commander args), add at minimum:
    - Test that `--exclude cursor,opencode` results in exclude array ['cursor', 'opencode']
    - Test that loadProviders is called with correct options including exclude

    IMPORTANT: The cli.ts file uses top-level await. Tests may need to handle this by mocking modules before dynamic import. Follow existing test patterns.
