<domain>
## Phase Boundary

Fix how cost estimates are communicated to Pro/Max subscribers, and refactor `pricing.ts` into a maintainable config structure. This phase does NOT add subscription detection or a settings UI — that is deferred. It delivers: honest relabelling, visible unknown-model warnings, and a split pricing config layer.

</domain>

<decisions>
## Implementation Decisions

### Cost labelling
- Keep the `$X.XX est.` suffix as-is in displayed values (no copy change to the numbers themselves)
- Add a small info icon (ⓘ) next to cost headings/labels (not every individual number) with a tooltip
- Tooltip text: "Based on API token pricing. If you're on Claude Pro or Max, your actual spend is your subscription fee — not these numbers."
- The tooltip should appear on: the Overview stat card labels ("All-time est.", "Selected period est."), the Models page total, and the Sessions column header
- Do NOT add a banner or footer note — tooltip is sufficient and lower noise

### Subscription awareness
- No subscription detection or user-declared plan in this phase
- The app remains "API-equivalent mode" for all users
- Defer subscription-aware suppression of dollar amounts to a future phase if user demand justifies it
- This decision is captured in the roadmap as "Recommendation: start with (A)" and we honor it

### Unknown model handling
- Currently: unknown models silently get `costUsd=0`, `unknownModel=true`, warning goes to stderr only
- New behavior: surface a visible UI indicator when any session/event has `unknownModel=true`
- Placement: a small warning badge or note in the Models page (e.g., "⚠ X sessions with unrecognized model IDs — costs may be understated")
- Do NOT block the user or show a modal — it's informational only
- The API already returns `unknownModel` on cost events; the UI just needs to consume it

### Pricing config structure
- Split `src/cost/pricing.ts` into two layers:
  1. **Tier definitions** — named constants for each pricing tier (e.g., `SONNET_4_PRICING`, `OPUS_4_PRICING`)
  2. **Model ID → tier mapping** — maps exact model ID strings to tier constant references
- Add a `lastUpdated` date field at the top of the config (already present as a comment — make it a constant)
- Add a `pricingSource` URL constant pointing to the Anthropic pricing page
- Keep the `ModelPricing` interface and `KnownModel` type in the same file or a companion types file
- No automated CI price drift check in this phase (deferred) — just better structure and the `lastUpdated` constant

### Claude's Discretion
- Exact file split (single file with two sections vs. `pricing-config.ts` + `pricing.ts` vs. other arrangement)
- Whether `lastUpdated` is a `const` string or a comment — as long as it's machine-readable
- Exact tooltip component implementation (custom vs. a library tooltip if one already exists)
- Exact placement of the ⓘ icon in the layout

</decisions>

<specifics>
## Specific Ideas

- Roadmap explicitly says: "honest relabelling is a low-risk improvement" — keep the change minimal
- The `$X.XX est.` format is already consistent everywhere; don't break that consistency
- Tooltip text should mention Pro and Max by name so users recognize their plan
- The unknown model warning is specifically for new models not yet in `pricing.ts` — Claude 4.7, Claude 5, etc.

</specifics>
