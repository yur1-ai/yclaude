  <files>.github/workflows/ci.yml</files>
  <action>
    Create the `.github/workflows/` directory and `ci.yml`:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      check:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: 24
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Install web dependencies
            run: cd web && npm ci

          - name: Lint (Biome)
            run: npx @biomejs/biome ci .

          - name: Type check
            run: npm run typecheck

          - name: Test
            run: npm test

          - name: Build
            run: npm run build
    ```

    Key details:
    - `npm ci` (not `npm install`) — deterministic, reads package-lock.json, faster in CI
    - `cd web && npm ci` — web has its own node_modules for Vite/React deps
    - `npx @biomejs/biome ci .` — the `ci` subcommand is the correct read-only command (NOT `check --no-fix` which is not a real flag)
    - `npm run build` (dev build, not prod) — validates the build pipeline without the slower bundling step
    - No matrix needed: ubuntu-latest + Node 24 only (per locked decision)
  </action>

<task type="auto">
  <name>Task 2: Create publish.yml — tag-triggered publish + release workflow</name>
  <files>.github/workflows/publish.yml</files>
  <action>
    Create `.github/workflows/publish.yml`:

    ```yaml
    name: Publish

    on:
      push:
        tags:
          - 'v*'

    jobs:
      check:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: 24
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Install web dependencies
            run: cd web && npm ci

          - name: Lint (Biome)
            run: npx @biomejs/biome ci .

          - name: Type check
            run: npm run typecheck

          - name: Test
            run: npm test

          - name: Build (production)
            run: npm run build:prod

          - name: Validate package contents
            run: |
              npm pack --dry-run 2>&1 | tee pack-output.txt
              grep -q "dist/server/cli.js" pack-output.txt || (echo "ERROR: dist/server/cli.js missing from package" && exit 1)
              grep -q "dist/web/index.html" pack-output.txt || (echo "ERROR: dist/web/index.html missing from package" && exit 1)
              echo "Package validation passed"

      publish:
        needs: check
        runs-on: ubuntu-latest
        permissions:
          contents: write
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: 24
              registry-url: 'https://registry.npmjs.org'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Install web dependencies
            run: cd web && npm ci

          - name: Build (production)
            run: npm run build:prod

          - name: Publish to npm
            run: npm publish
            env:
              NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

          - name: Create GitHub Release
            uses: softprops/action-gh-release@v2
            with:
              generate_release_notes: true
    ```

    Critical details to get right:
    - `needs: check` on the `publish` job — publish NEVER runs if check fails; this is the safety gate
    - `registry-url: 'https://registry.npmjs.org'` in the `publish` job's `setup-node` — REQUIRED for `NODE_AUTH_TOKEN` to be written to `.npmrc`; without this the publish step silently fails with auth error
    - `permissions: contents: write` on `publish` job — required for `softprops/action-gh-release` to create GitHub Releases
    - `generate_release_notes: true` — GitHub auto-generates notes from commits since last tag
    - `GITHUB_TOKEN` is implicitly available for `softprops/action-gh-release` — no additional secret needed
    - The `check` job uses `npm run build:prod` (not `npm run build`) so the full prod validation runs before publish
    - `npm pack --dry-run` validation in `check` job catches packaging mistakes before publish

    Note for future: NPM_TOKEN must be set as a GitHub repo secret (Settings > Secrets > Actions) before the first tag push. The token must be a granular access token with "Bypass two-factor authentication" enabled (classic tokens were revoked December 2025).
