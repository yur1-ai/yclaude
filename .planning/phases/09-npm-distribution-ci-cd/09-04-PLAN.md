<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Package is built and verified locally. All automated checks pass:
    - `npm run build:prod` produces `dist/` with bundled deps
    - `npm pack --dry-run` confirms correct tarball contents
    - Server starts and serves HTML from `dist/web/`

    The following manual steps require human action — Claude cannot create npm accounts, store GitHub secrets, or log into npm.
  </what-built>
  <how-to-verify>
    Complete these steps in order:

    **Step 1: Confirm npm name availability**
    - Visit https://www.npmjs.com/package/yclaude
    - If it shows "404 Not Found" or "Package not found" — name is available, proceed
    - If a package exists — stop and report; we need a new name decision

    **Step 2: Create npm granular access token**
    - Go to https://www.npmjs.com/settings/~/tokens
    - Click "Generate New Token" → "Granular Access Token"
    - Set name: `yclaude-ci-publish`
    - Set expiration: 90 days (maximum allowed for write tokens)
    - Under "Packages and scopes" → select "yclaude" package (or "All packages" if it doesn't exist yet)
    - Set permissions: Read and write
    - Check "Bypass two-factor authentication" (required for CI — classic tokens no longer exist)
    - Click "Generate token" and copy it (shown only once)

    **Step 3: Add NPM_TOKEN to GitHub secrets**
    - Go to your GitHub repo → Settings → Secrets and variables → Actions
    - Click "New repository secret"
    - Name: `NPM_TOKEN`
    - Value: paste the token from Step 2
    - Click "Add secret"

    **Step 4: First manual publish**
    Run from the project root:
    ```bash
    npm publish
    ```
    This will trigger `prepublishOnly` which runs `build:prod` first.
    You will be prompted to authenticate if not already logged in via `npm login`.

    **Step 5: Verify live on npmjs.com**
    - Wait 30-60 seconds after publish
    - Visit https://www.npmjs.com/package/yclaude
    - Confirm: version 0.1.0 shows, README renders, install command appears

    **Step 6: Smoke test cold install (optional but recommended)**
    In a temp directory on your machine:
    ```bash
    cd /tmp && mkdir yclaude-test && cd yclaude-test
    npx yclaude --version
    ```
    Should print `0.1.0` without errors.
  </how-to-verify>
  <resume-signal>Type "published" when yclaude 0.1.0 is live on npmjs.com, or describe any issues encountered.</resume-signal>
</task>
