    1. **Content extraction helpers** (add in api.ts or a separate utility):

    ```typescript
    interface ContentBlock {
      type: 'text' | 'tool_use' | 'tool_result';
      text?: string;
      toolName?: string;
      toolInput?: Record<string, unknown>;
      toolId?: string;
      toolUseId?: string;
      resultContent?: string;
      isError?: boolean;
    }
    ```

    Create `extractContentBlocks(message: Record<string, unknown>): ContentBlock[]`:
    - If `message.content` is a string, return `[{ type: 'text', text: content }]`
    - If array, iterate blocks:
      - `type: 'text'` -> `{ type: 'text', text: block.text }`
      - `type: 'tool_use'` -> `{ type: 'tool_use', toolName: block.name, toolInput: block.input, toolId: block.id }`
      - `type: 'tool_result'` -> `{ type: 'tool_result', toolUseId: block.tool_use_id, resultContent: truncate(block.content, 10000), isError: block.is_error }`
      - `type: 'thinking'` -> SKIP (filter out)
    - Return array of ContentBlock

    Create `extractFirstUserMessage(events: NormalizedEvent[]): { truncated: string; full: string }`:
    - Find first event with `type === 'user'` and a message field
    - Extract text content from the message
    - Return `{ truncated: text.slice(0, 80) + (text.length > 80 ? '...' : ''), full: text }`
    - Return `{ truncated: '', full: '' }` if no user message found

    Create `getTextContent(events: NormalizedEvent[]): string`:
    - Concatenate all text content blocks from all events (for search matching)
    - Used by search filter

    2. **GET /api/v1/chats** endpoint:
    - Gate: `if (!state.showMessages || !state.rawEvents) return c.json({ error: 'Conversation viewing is disabled. Start with --show-messages to enable.' }, 403)`
    - Parse query params: from, to, project, search, page (same pattern as /sessions)
    - Group `state.rawEvents` by sessionId
    - For each session group:
      - Extract first user message: `const { truncated, full } = extractFirstUserMessage(sessionEvents)`
      - Build chat list item: sessionId, cwd, displayName (using assignProjectNames), model (from first assistant event), costUsd (sum), timestamp (first event), firstMessage: truncated, firstMessageFull: full, turnCount (count of user+assistant events)
    - Apply search filter: case-insensitive substring match on `getTextContent(events)` -- the search string must appear somewhere in user or assistant text content
    - Sort by timestamp descending (newest first per user decision)
    - Paginate: 50 per page
    - Return `{ chats, total, page, pageSize: 50 }`

    3. **GET /api/v1/chats/:id** endpoint:
    - Gate: same 403 check
    - Filter rawEvents by sessionId
    - 404 if no events found
    - Sort events by timestamp ascending
    - Build thread: for each event, extract role (type), content blocks, timestamp, model (for assistant), tokens (for assistant)
    - Build summary (mirror SessionDetail pattern): sessionId, displayName, cwd, model, models, totalCost, totalTokens, timestamp, durationMs, gitBranch
    - Return `{ summary, messages }` where messages is the ordered array of turns with content blocks
