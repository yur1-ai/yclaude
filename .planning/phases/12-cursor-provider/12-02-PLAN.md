  <action>
**Replace the stub adapter.ts** with full implementation:

`detect()`:
1. Call `getCursorDataDirs()` to get candidate directories (or use `this.testDataDirs` if provided).
2. For each directory, check if `getGlobalDbPath(dir)` exists using `fs.access()`.
3. Return true if ANY global state.vscdb exists.
4. Wrap in try/catch -- return false on any error.

`load(opts: LoadOptions)`:
1. If `opts.debug`, call `enableDebug()`.
2. Call `getCursorDataDirs()` (or `this.testDataDirs`), filter to existing directories (check global DB exists).
3. For each valid data dir:
   a. Get `workspaceStoragePath`.
   b. Read directory listing of workspace storage (each subdirectory is a workspace).
   c. For each workspace subdirectory:
      - Read `workspace.json` if it exists. Parse `folder` field (URL-encoded path like `file:///path/to/project`). Decode to plain path.
      - Check if `state.vscdb` exists in the workspace dir.
      - Add to workspaces array: `{ dbPath: wsDbPath, workspacePath: decodedFolder }`.
   d. Get `globalDbPath`.
   e. Open global DB briefly to detect schema version. Log to stderr: `[yclaude] Cursor: schema v${version} (${count} bubbles sampled)`. Close DB.
4. Call `parseCursorData({ globalDbPath, workspaces, preserveContent: opts.preserveContent, debug: opts.debug })`.
5. Return the UnifiedEvent[] result.
6. All file I/O in try/catch with meaningful error messages.

Use `fs.access()` for existence checks (not `fs.stat` -- access is lighter).
Use `fs.readdir()` for workspace directory listing.
Use `fs.readFile()` for workspace.json.
Decode `file:///` URLs: strip `file://` prefix, then `decodeURIComponent()`.
Import from node:fs/promises for all async file operations.

For testability, the adapter accepts an optional `dataDirs` parameter in the constructor: `constructor(private testDataDirs?: string[])`. When provided, `detect()` and `load()` use these instead of `getCursorDataDirs()`. This avoids mocking os.homedir and makes tests deterministic. The registry still creates `new CursorAdapter()` with no args for production.
