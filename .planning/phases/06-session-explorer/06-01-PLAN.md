<task type="auto" tdd="true">
  <name>Task 1: GET /api/v1/sessions — paginated session list endpoint</name>
  <files>src/server/routes/api.ts</files>
  <behavior>
    - With no filters: returns all sessions grouped from state.costs, sorted by timestamp descending
    - With ?page=2: returns sessions 51-100 (page size is always 50)
    - With ?project=/some/cwd: returns only sessions where cwd matches that string
    - With ?from= and ?to= ISO strings: filters events by timestamp before grouping
    - Sessions with no token-bearing events (e.tokens undefined) are excluded from results
    - durationMs is the maximum value found across ALL events in the session (not just token events); null if no event has durationMs
    - model field is the single model name if only one distinct model used, otherwise "Mixed"
    - models field is array of all distinct model strings (for tooltip)
    - displayName comes from assignProjectNames() called on all unique cwds
    - cwd for a session is the cwd of the earliest event (by timestamp)
    - timestamp is the ISO string of the earliest event in the session
    - Response shape: { sessions: SessionRow[], total: number, page: number, pageSize: number }
    - Invalid ?from or ?to returns 400 with { error: string }
  </behavior>
  <action>
    Replace the stub `app.get('/sessions', (c) => c.json({ sessions: [] }))` with a full implementation.

    Add a TypeScript interface at the top of the route block (not exported — local to apiRoutes):
    ```typescript
    interface SessionRow extends Record<string, unknown> {
      sessionId: string;
      displayName: string;
      cwd: string | null;
      model: string;
      models: string[];
      costUsd: number;
      tokens: { input: number; output: number; cacheCreation: number; cacheRead: number };
      timestamp: string;
      durationMs: number | null;
    }
    ```

    Implementation steps:
    1. Parse query params: `from`, `to` via `parseDate()`, `project` (nullable string), `page` (parseInt, default 1, min 1)
    2. Return 400 if parseDate returns 'invalid'
    3. Filter `state.costs` by from/to date range (compare `new Date(e.timestamp).getTime()`)
    4. If projectFilter set, further filter by `(e.cwd ?? null) === projectFilter`
    5. Group filtered events by `sessionId` using `Map<string, { events: CostEvent[] }>` — collect ALL events per session (not just token-bearing)
    6. For each session group, build SessionRow:
       - `sessionId`: the key
       - Sort events by timestamp ascending (use `.getTime()` comparison, stable by original index)
       - `timestamp`: earliest event's timestamp (ISO string)
       - `cwd`: earliest event's `cwd ?? null`
       - `durationMs`: `Math.max` across all events' durationMs values, or null if none have it
       - Token-bearing events = events where `e.tokens !== undefined`
       - SKIP session entirely if it has ZERO token-bearing events
       - `costUsd`: sum of `e.costUsd` across ALL events
       - `tokens`: sum of input/output/cacheCreation/cacheRead across token-bearing events only
       - `models`: `[...new Set(token-bearing events map e.model ?? 'Unknown')]`
       - `model`: models.length === 1 ? models[0] : 'Mixed'
    7. Collect all unique cwds from sessions, call `assignProjectNames(cwds)`, set `displayName` on each row
    8. Sort all sessions by timestamp descending (`new Date(b.timestamp).getTime() - new Date(a.timestamp).getTime()`)
    9. Slice: `total = sessions.length`, `sessions = allSessions.slice((page-1)*PAGE_SIZE, page*PAGE_SIZE)`
    10. Return `c.json({ sessions, total, page, pageSize: PAGE_SIZE })`

    PAGE_SIZE constant = 50. Define as `const PAGE_SIZE = 50;` inside the route handler.

    CRITICAL: Never include any `message`, `content`, or prose text fields in SessionRow. Only structured metadata.
    CRITICAL: Use `e.costUsd` directly (it serializes as a plain number despite the EstimatedCost brand).
  </action>

  </verify>

<task type="auto" tdd="true">
  <name>Task 2: GET /api/v1/sessions/:id — session detail endpoint</name>
  <files>src/server/routes/api.ts</files>
  <behavior>
    - Valid sessionId returns { summary: SessionSummary, turns: TurnRow[] }
    - turns are sorted by timestamp ascending (stable: original array order preserved for ties)
    - each turn has: turn (1-indexed), model, tokens (input/output/cacheCreation/cacheRead), costUsd, cumulativeCost (running total), timestamp
    - cumulativeCost on turn N = sum of costUsd for turns 1..N
    - summary contains: sessionId, displayName, cwd, model ("Mixed" if multiple), models[], totalCost (sum ALL events), totalTokens (sum token-bearing events), timestamp (earliest), durationMs (max across ALL events or null), gitBranch (first non-null gitBranch across all events or null)
    - Returns 404 with { error: 'Session not found' } if no events exist for that sessionId
    - No message/content/prose fields appear anywhere in the response
  </behavior>
  <action>
    Add after the `/sessions` route:
    ```
    app.get('/sessions/:id', (c) => { ... })
    ```

    Add local TypeScript interfaces (not exported):
    ```typescript
    interface TurnRow {
      turn: number;
      model: string;
      tokens: { input: number; output: number; cacheCreation: number; cacheRead: number };
      costUsd: number;
      cumulativeCost: number;
      timestamp: string;
    }

    interface SessionSummary {
      sessionId: string;
      displayName: string;
      cwd: string | null;
      model: string;
      models: string[];
      totalCost: number;
      totalTokens: { input: number; output: number; cacheCreation: number; cacheRead: number };
      timestamp: string;
      durationMs: number | null;
      gitBranch: string | null;
    }
    ```

    Implementation steps:
    1. `const sessionId = c.req.param('id')`
    2. `const allEvents = state.costs.filter((e) => e.sessionId === sessionId)`
    3. If `allEvents.length === 0`, return `c.json({ error: 'Session not found' }, 404)`
    4. Sort allEvents by timestamp ascending (stable by original index for ties — JS .sort() is stable in V8)
    5. Token-bearing events = `allEvents.filter((e) => e.tokens !== undefined)`
    6. Build turns array from token-bearing events:
       - Map with running `cumulative` variable starting at 0
       - Each turn: `{ turn: i+1, model: e.model ?? 'Unknown', tokens: { input: e.tokens.input, output: e.tokens.output, cacheCreation: e.tokens.cacheCreation, cacheRead: e.tokens.cacheRead }, costUsd: e.costUsd, cumulativeCost: (cumulative += e.costUsd), timestamp: e.timestamp }`
       - EXPLICIT field construction — never spread the raw event
    7. Build summary from allEvents:
       - `cwd`: allEvents[0].cwd ?? null (earliest, already sorted)
       - `timestamp`: allEvents[0].timestamp
       - `durationMs`: all events reduce to find max durationMs (or null)
       - `gitBranch`: allEvents.find(e => e.gitBranch)?.gitBranch ?? null
       - `totalCost`: allEvents.reduce sum of costUsd
       - `models`: [...new Set(token-bearing events map e.model ?? 'Unknown')]
       - `model`: models.length === 1 ? models[0] : 'Mixed'
       - `totalTokens`: reduce over token-bearing events summing input/output/cacheCreation/cacheRead
       - `displayName`: call `assignProjectNames([cwd]).get(cwd) ?? cwd ?? 'Unknown'`
    8. Return `c.json({ summary, turns })`
  </action>
