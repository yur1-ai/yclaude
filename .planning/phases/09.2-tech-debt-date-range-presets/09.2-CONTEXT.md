<decisions>
## Implementation Decisions

### Pricing tooltip content
- Add a footer line below existing tooltip text: "Pricing last verified: {PRICING_LAST_UPDATED}" with human-readable date format (e.g., "Feb 28, 2026")
- Keep existing Pro/Max explanation copy as-is, append footer
- PRICING_LAST_UPDATED and PRICING_SOURCE imported at build time directly from `src/cost/pricing.ts` (no API endpoint, no shared file — tree-shaking drops everything else)
- The ⓘ icon itself becomes a clickable `<a>` tag opening PRICING_SOURCE in a new tab
- Affordance: `cursor-pointer` + hover color change on icon to signal clickability
- Tooltip text includes a "Click ⓘ to view source" hint line
- Tooltip stays CSS-only hover (no JS state, no pin/delay mechanics)

### Period cost trend
- Wire usePriorSummary into Overview's period cost StatCard (already proven in CacheEfficiencyCard)
- Show trend for named presets only: 24h, 48h, 7d, 30d, 90d
- No trend for "All time" (no prior period) or "Custom" (arbitrary ranges)
- When prior period has no data (new user, insufficient history): show nothing (TrendIndicator hidden, not "No prior data" text)
- When prior period cost is $0: hide trend (division by zero — same as no data)
- Suppress changes below ±1% — prevents distracting micro-fluctuations
- Percentage only (no absolute dollar difference) — keeps StatCard compact
- Color: green arrow for spending down, red arrow for spending up — matches cost-saving mental model
- Trend appears on period StatCard only, not All-time StatCard
- Fire usePriorSummary immediately on preset change (no debounce) — local server, TanStack Query caches, proven pattern from CacheEfficiencyCard

### Date range presets
- Chronological order: 24h | 48h | 7d | 30d | 90d | All time | Custom
- Short labels matching existing style: "24h", "48h"
- Active label: "Last 24h", "Last 48h" — matches existing "Last 7d" pattern
- Rolling window: 24h = now minus 24 hours (not calendar day); same for 48h
- Selecting 24h or 48h auto-switches chart bucket to hourly (not just enables the button)
- Switching FROM 24h/48h TO a wider preset (7d+) auto-reverts bucket to 'day' — symmetric behavior (existing useEffect already handles this)
- All preset buttons styled identically — no visual distinction for hourly-capable presets
- Seven buttons — let flexbox wrap naturally on narrow screens (simplest approach)
- Update Preset type in useDateRangeStore.ts and getActiveLabel in DateRangePicker.tsx

### Dead code removal
- Remove /api/v1/events stub (returns empty array, no consumer)
- Remove QUIPS.loading_generic (defined but never referenced in any component)

### Claude's Discretion
- Flexbox wrap breakpoint details
- Whether auto-bucket-switch uses useEffect or callback
- Exact tooltip text wording refinements (within the decided structure)

</decisions>
