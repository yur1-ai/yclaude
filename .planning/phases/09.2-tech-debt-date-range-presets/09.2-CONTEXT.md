<decisions>
## Implementation Decisions

### Pricing tooltip content
- Add a footer line below existing tooltip text: "Pricing data last verified: {PRICING_LAST_UPDATED}" with clickable source link
- Keep existing Pro/Max explanation copy as-is, append footer
- PRICING_LAST_UPDATED and PRICING_SOURCE imported at build time from shared module (no API endpoint needed — these are compile-time constants)
- Source URL rendered as a real clickable `<a>` tag so users can verify pricing

### Period cost trend
- Wire usePriorSummary into Overview's period cost StatCard (already proven in CacheEfficiencyCard)
- Show trend for named presets only: 24h, 48h, 7d, 30d, 90d
- No trend for "All time" (no prior period) or "Custom" (arbitrary ranges)
- When prior period has no data (new user, insufficient history): show nothing (TrendIndicator hidden, not "No prior data" text)
- Color: green arrow for spending down, red arrow for spending up — matches cost-saving mental model
- Trend appears on period StatCard only, not All-time StatCard

### Date range presets
- Chronological order: 24h | 48h | 7d | 30d | 90d | All time | Custom
- Short labels matching existing style: "24h", "48h"
- Selecting 24h or 48h auto-switches chart bucket to hourly (not just enables the button)
- Seven buttons — let flexbox wrap naturally on narrow screens (simplest approach)
- Update Preset type in useDateRangeStore.ts and getActiveLabel in DateRangePicker.tsx

### Dead code removal
- Remove /api/v1/events stub (returns empty array, no consumer)
- Remove QUIPS.loading_generic (defined but never referenced in any component)

### Claude's Discretion
- Exact shared module path for pricing metadata import (could be direct import from server src/cost/pricing.ts or a shared constants file)
- TrendIndicator threshold for showing 0% vs hiding (e.g., ±0.1% change)
- Flexbox wrap breakpoint details
- Whether auto-bucket-switch uses useEffect or callback

</decisions>
