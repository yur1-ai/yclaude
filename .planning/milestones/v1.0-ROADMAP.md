# Milestone v1.0: Local MVP

**Status:** ✅ SHIPPED 2026-02-28
**Phases:** 1–4
**Total Plans:** 11

## Overview

yclaude v1.0 delivers the complete local foundation: a resilient JSONL parser, accurate cost engine, secure local server, and a working cost analytics dashboard — all launched via `npx yclaude`. The milestone validates the core product hypothesis: developers will use a zero-install CLI tool to get visibility into their AI coding spend.

## Milestone Summary

**Key Decisions:**
- Hash routing (createHashRouter) — works with static file serving, no server catch-all needed
- API routes registered before serveStatic — static catch-all would block API if first
- webDistPath via fileURLToPath(new URL) — npx correctness from dist/server/server.js
- UTC Date methods throughout bucketing (getUTCDay, setUTCDate) — local methods cause timezone drift
- parseDate() three-state return (null | Date | 'invalid') — distinguishes absent from malformed without boolean flags
- Zustand (not URL params) for global date filter — URL sync deferred to v1.x
- queryKey uses from?.toISOString() from store (not new Date()) — prevents infinite refetch loops
- react-day-picker CSS imported in DateRangePicker.tsx only — avoids global CSS pollution
- npm used for web/ deps — package-lock.json already present, npm was the configured manager

**Tech Debt Incurred:**
- TrendIndicator shows null % for all Phase 4 presets — prior-period query deferred to Phase 5+
- Proportional cost per token type is approximate (totalCost × typeShare) — exact per-type rates deferred to Phase 5+
- CostBarChart.tsx:78,85 — tick colors use `oklch(0.55 0 0)` inline (not CSS var); harmless info-only
- KnownModel type not re-exported from src/index.ts public surface — fix when Phase 5 needs per-model type safety
- AppState.events field populated but unused by any route — dead weight until /api/v1/events (Phase 6)
- --debug not declared as Commander option in cli.ts — auto-detects via process.argv

**Issues Resolved:**
- Phase 2 VERIFICATION flagged CORE-04 as partial (correct: server/CSP not yet built); resolved fully in Phase 3

**Issues Deferred:**
- Subagent token accounting confirmation (additive vs separate) — needed before Phase 7
- react-activity-calendar theming with Tailwind v4 CSS vars — proof-of-concept needed before Phase 7
